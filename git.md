## 1.6. Git分支

### **1.6.1.** ***\*分支在实际开发中的作用\****

在进行多人协作开发的时候，为了防止互相干扰，提高协同开发的体验，建议每个开发者都基于分支进行项目功能的开发







### **1.6.10.** ***\*遇到冲突时的分支合并\****

如果在两个不同的分支中，对同一个文件进行了不同的修改，Git就没法干净的合并它们。此时，我们需要打开这些包含冲突的文件然后手动解决冲突

解决方式为：

打开包含冲突的文件，手动解决冲突之后，再执行如下的命令

git add .

git commit -m+”注释内容”
